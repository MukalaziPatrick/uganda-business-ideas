## Next Session — UI/UX Polish Agenda

The app works but needs visual polish before going live. Here's the prioritised list:

### Priority 1 — Homepage
- Hero needs to feel more Ugandan / less generic SaaS. Consider: flag colors, local photography placeholder, Luganda tagline option
- Three pillars are stacked vertically (grid-cols-1) — on desktop should be side-by-side (grid-cols-3)
- Jobs teaser cards are too plain — add Apply button directly on homepage cards
- Nav is minimal — add mobile hamburger menu
- Footer is bare — add links: Post a Job, Register as Worker, Find Businesses

### Priority 2 — Jobs Page
- Hero gradient is good but search inputs need better contrast on mobile
- Job cards could show a category color dot/icon instead of text-only skill badge
- Worker profile cards in the tab are dense — needs breathing room on mobile
- Empty state illustrations (simple SVG or emoji art) to make zero-data states feel intentional
- Add "share" button on job cards (copy link)

### Priority 3 — Forms (Post Job + Worker Registration)
- Progress indicator / step labels so user knows how long the form is
- Success screen needs a "Share your profile" WhatsApp button
- Field validation error messages need to be more prominent (currently small red text)

### Priority 4 — Worker Profile Page
- Add a "Back to workers" breadcrumb link
- Profile page has no share button — add "Share via WhatsApp" to let workers share their own profile
- If no contact info at all, show a friendly fallback message instead of empty contact section

### Priority 5 — Mobile overall
- Test all pages on 375px viewport (iPhone SE size) — forms especially
- Sticky bottom CTA bar on homepage ("Browse Jobs ↗") for mobile users who scroll past hero

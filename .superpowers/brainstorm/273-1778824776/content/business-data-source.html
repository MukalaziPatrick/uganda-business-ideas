Where does business data come from?

How should real businesses (restaurants, plumbers, shops) surface on the site? Pick the approach that fits best.

A

New businesses table here ⭐ Recommended

Add a businesses table to uganda-business-ideas Supabase. Businesses register and are searched entirely on this site.

Pros

  • One codebase to maintain
  • Full control over schema & UI
  • Business registration form lives here
  • Search + map on same page

Cons

  • Duplicate data if same business is in LocateUG
  • Have to re-enter existing LocateUG businesses
B

Cross-project query from LocateUG

Query the existing LocateUG Supabase project remotely. Both sites share one businesses database.

Pros

  • No data duplication
  • Existing LocateUG businesses appear immediately

Cons

  • Exposes LocateUG's anon key in this app
  • Tight coupling between two separate apps
  • Two Supabase projects to manage
C

Deep-link to LocateUG

Keep LocateUG as source of truth. Searching "restaurants in Kampala" bounces you to LocateUG with filters pre-applied.

Pros

  • Zero backend work on this site
  • LocateUG stays the map authority

Cons

  • Users leave this site — jarring experience
  • No control over LocateUG's UI or data
  • Search box here feels hollow
